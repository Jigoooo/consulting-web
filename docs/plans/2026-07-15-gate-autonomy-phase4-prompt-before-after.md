## 1. 제안하는 변경 — 최소 2건 (창원 토픽 7개 공통)

### 변경 A — 중복 수치 hedge 2줄을 자가검증 블록으로 통합
현재 서로 다른 두 위치에 흩어진 동일 취지:
```
[(4)③ 아래]  - 수치 단정은 ①(검증자료) 또는 ②(실제 출처 확인) 근거가 있을 때만. 투자성 단정 금지. ...
[원칙 아래]   - 수치는 extracted_values 근거가 있을 때만 단정. 근거 없으면 정성적으로만 서술하고 '데이터 확인 제한' 표기.
```
→ **통합안**(한 곳, permission-forward 톤):
```
- 수치: 검증자료(extracted_values) 또는 실제 확인한 출처가 있으면 근거와 함께 명확히 단정한다. 근거가 없을 때만 정성적으로 서술하고 '데이터 확인 제한'을 표기한다. 투자성 단정은 하지 않는다.
```
효과: 반복 2회 → 1회. 톤을 "~하지 마"에서 "근거 있으면 단정하라"로 전환(deliberative). 안전(투자성 단정 금지)은 유지.

### 변경 B — 3층 답변 구조의 톤 미세 조정 (선택)
현재 `(4)①` "있으면 근거로 단정 인용"은 이미 permission-forward라 유지. `③` "단순 나열 금지"만 `③ 지구의 분석·해석·의견을 반드시 결합한다(사실과 추정은 말로 구분)` 로 긍정형 정리.
