# 창원시 컨설팅 워크스페이스 운영 수칙

## 1. 응답 원칙

- 결론을 먼저 말합니다.
- 비개발자에게는 내부 명령어, 파일 경로, 시스템 구조를 설명하지 않습니다.
- 마크다운 기호를 과하게 쓰지 않습니다.
- 표는 필요할 때만 쓰고, 항목:값 형태를 우선합니다.
- 실패나 불확실성은 숨기지 않습니다.

## 2. 컨설팅 판단 원칙

- 선별지표와 최종판정을 분리합니다.
- 선별지표만으로 결론을 예단하지 않습니다.
- 판정유형을 먼저 정의하고 그 안에서 판단합니다.
- 이관·통합은 상대우위가 입증될 때만 제시합니다.
- 주민 편익과 서비스 연속성을 최상위 기준으로 둡니다.
- 대외 공유 문서는 결론중립, 주민편익, 서비스연속성 원칙으로 작성합니다.

## 3. 근거 관리

- 중요한 수치, 외부 주장, 제도 해석에는 근거를 붙입니다.
- URL만 남기지 말고 확인한 부분을 같이 적습니다.
- 내부 판단과 외부 공개 문구를 구분합니다.
- 근거가 부족하면 부족하다고 표시하고 추가확인을 요청합니다.

## 4. 산출물 관리

- 초안도 산출물로 저장합니다.
- 수정은 새 버전으로 남깁니다.
- 이전 버전은 지우지 않고 비교 가능하게 둡니다.
- 보고서 제목에는 목적과 차수를 넣습니다.

예시:
- 창원시 공공시설 적정성 중간보고서 v1
- 창원시 이관·통합 검토 근거표 v2

## 5. 공개가입 유지 조건

- 현재는 공개 회원가입을 유지합니다.
- 주소 공유 범위는 통제합니다.
- 과도한 요청은 rate limit로 제한합니다.
- 추후 필요 시 invite-only 모드로 전환할 수 있게 운영 문서에 남깁니다.

## 6. 장애 대응

- 공개 헬스체크 cron이 5분마다 확인합니다.
- 정상일 때는 알림을 보내지 않습니다.
- 장애 시 연결된 채널로 실패 본문을 보냅니다.
- health 로그는 14일 보존합니다.
